## Scope boundaries confirmed

The R23 MVP is intentionally narrow. The following are explicitly in scope:

- one site
- one sector
- three UEs
- transmit-side EIRP only

The following are explicitly out of scope and are not implemented in the MVP:

- no path loss
- no clutter
- no FS / FSS receiver modeling
- no interference aggregation
- no 19-site or 57-sector simulation

## Known limitations

- The MVP exposes per-direction transmit-side EIRP only. CDFs are over
  UE-driven beam pointings, not over time or propagation.
- The R23 extended path is not bit-equivalent to pycraf; only the simple
  M.2101 path is gated by `test_against_pycraf_strict` at 1e-6 dB.
- Pycraf comparison tests skip cleanly when Python or pycraf is not
  available; skipped tests count as pass for the overall summary.
- The full per-snapshot EIRP cube returned by `run_monte_carlo_snapshots`
  is intentional for the BS-driven MVP. For large grids prefer the
  streaming `runR23AasEirpCdfGrid` runner.
- This is **not ITU-certified**. Cross-check against M.2101 and 3GPP
  TR 37.840 before using results in regulatory contexts.
